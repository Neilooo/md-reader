# 更新日志

## [Unreleased]

## [0.3.2] - 2026-07-21

### 新增
- 设置弹窗新增"快捷键"面板：点击"查看快捷键"按钮可查看所有快捷键，支持自定义绑定。点击键帽进入录制态，按下新的组合即可更改；冲突时自动检测并提示冲突项；支持逐项重置和全部重置。自定义快捷键持久化保存，重启后保留。
- 全局快捷键和 CodeMirror 编辑器快捷键（加粗、斜体、下划线、高亮、行内代码等）均可自定义。
- 工具栏按钮 tooltip 显示当前生效的快捷键，自定义后自动更新。

## [0.3.1] - 2026-07-21

### 新增
- 设置弹窗新增"大纲位置"选项（左侧/右侧）：选择左侧时大纲作为左侧栏 tab 显示、工具栏大纲按钮隐藏；选择右侧时大纲在右侧独立面板、左侧栏仅显示文件和搜索。默认右侧。
- 左侧栏新增"大纲"tab（Files | Search | Outline），可将大纲放在左侧使用；右侧独立大纲面板保留，两者可同时显示，各自独立折叠状态。
- 大纲支持分级展开折叠：有子标题的标题显示 ▶/▼ 箭头点击切换，顶部提供"全部展开/全部折叠"按钮；切换文档时自动重置折叠状态。

## [0.3.0] - 2026-07-20

### 新增
- 导出 PDF/HTML/DOCX 时，保存对话框默认路径改为源文件同目录同名文件（如 `abc.md` 导出 PDF 默认为同目录 `abc.pdf`）。感谢 @openxin2023 贡献（#13）。

## [0.2.9] - 2026-07-13

### 新增
- 编辑模式增加 `Ctrl+鼠标滚轮` 缩放字体，与现有 `Ctrl+=/-` 快捷键并存，两模式分流（编辑模式调编辑器字号，预览模式调阅读字号）。
- 新增 `Ctrl+O` 打开文件快捷键。
- 空状态区域展示最近使用文件列表：无文件打开时自动列出最近 10 个文件，点击直接打开；展示前用 `exists()` 预检查文件存在性，已删除或移动的文件自动过滤；提供"清除列表"按钮。

### 修复
- 修复编辑模式下选中文字高亮不明显的问题：选中色改为主题感知（浅色模式 `rgba(9,105,218,0.3)`，深色模式 `rgba(100,160,255,0.3)`），在当前行背景上清晰可辨。

## [0.2.8] - 2026-07-10

### 新增
- 编辑模式支持直接粘贴图片：在编辑模式下 `Ctrl+V` 粘贴截图或剪贴板图片，自动保存到当前 Markdown 文件同目录的 `images/` 文件夹，并在光标位置插入 Markdown 图片链接。支持 PNG/JPEG/GIF/WebP/BMP 格式。

### 修复
- 修复编辑模式下点击导出 PDF/HTML/DOCX 或打印时，红色错误提示会替换整个编辑器导致界面卡死、无法切回预览模式的问题。错误提示现在改为顶部可关闭的横幅，不再遮挡编辑器和预览区，5 秒后自动消失。

## [0.2.7] - 2026-07-08

### 新增
- 编辑模式支持自主放大/缩小字体：快捷键 `Ctrl+=` 放大、`Ctrl+-` 缩小、`Ctrl+0` 重置，两模式分流（编辑模式调编辑器字号，预览模式调阅读字号）。
- 设置弹窗新增"编辑器字号"滑块（12-24px），与阅读字号滑块并列，可精确调节。
- 编辑器字号持久化记忆，重启后保留。

### 变更
- 预览/编辑模式切换快捷键从 `Ctrl+/` 改为 `Ctrl+E`。

## [0.2.6] - 2026-07-06

### 修复
- 修复点击"打印 / 系统 PDF"无任何反应的问题：Tauri 2 的 WebView2 会拦截 `window.open`，导致原打印流程静默失败。改为通过隐藏 iframe 承载打印内容并调用 `contentWindow.print()` 弹出系统打印对话框，不再依赖 `window.open`。
- 补充打印流程的错误处理，构建打印 HTML 失败时不再被静默吞掉。

## [0.2.5] - 2026-07-03

### 新增
- 设置弹窗新增“Word 导出模板”选项：可选择一个 `.docx` 作为 pandoc 样式参考文档（`--reference-doc`），导出 DOCX 时静默使用该模板控制字体、标题和段落样式；未设置时按默认方式导出，行为不变。

## [0.2.4] - 2026-07-02

### 修复
- 修复 Mermaid 流程图节点文字不显示的问题：SVG 输出不再经过 HTML sanitize 清洗，改为在 SVG DOM 中仅剥除 `<script>` 与事件/`javascript:` 属性，保留节点文本结构。

## [0.2.2] - 2026-06-30

### 新增
- 新增 YAML Front Matter 解析与预览：Markdown 顶部 `---` 元数据会显示为 Front Matter 信息卡片，正文和大纲会剥离元数据后渲染。
- 编辑模式新增 Markdown 格式快捷键：`Ctrl+B` 加粗、`Ctrl+I` 斜体、`Ctrl+U` 下划线、`Ctrl+L` 高亮、`Ctrl+Shift+\`` 行内代码。
- 新增 `Ctrl+/` 在预览与编辑模式之间快速切换。
- 设置页新增 Windows 当前用户文件关联注册入口，将 MD Reader 写入 `.md / .markdown / .mdx` 的“打开方式”列表。
- 新增宽表格测试文档，便于验证超宽表格横向滚动表现。

### 优化
- 导出 HTML/PDF/DOCX 对话框标题和阅读设置字体选项支持中英文切换。
- Mermaid SVG 渲染结果增加 DOMPurify 清理，错误信息改为安全文本写入。
- 导出远程图片内联时阻止请求本机和私有网络地址。
- 打印改用 Blob URL 打开临时页面，减少 `document.write()` 带来的阻塞和兼容问题。
- 限制历史滚动位置缓存数量，避免长期使用后本地存储无限增长。
- 预览/编辑模式切换时按 Markdown 源码行同步视口位置，减少长文档切换后位置跳变。

### 修复
- 修复全局搜索结果高亮在包含特殊 HTML 字符时的转义和匹配处理。
- 修复数学公式解析中 `$   $` 这类纯空白内容未被过滤的问题。
- 修复文件树同名目录可能共用折叠状态的问题。
- 修复超宽表格在阅读区内被截断的问题，现在表格会通过包装层横向滚动。
- 修复右侧大纲目录内容过长时没有滚动条、下方条目被裁剪的问题。
- 修复通过文件关联或右键“打开方式”启动时可能仍显示上次阅读文件的问题：直接启动仍恢复上次会话，通过文件启动时会优先激活传入文件。

## [0.2.1] - 2026-06-29

- 编辑模式打开查找时自动聚焦并选中搜索输入框，点击工具栏查找或按 `Ctrl+F` 后可直接输入关键词。
- 编辑模式新增查找匹配计数浮层，显示类似 `1/4` 的当前匹配位置和总匹配数。
- 编辑模式查找计数改为节流和流式统计，减少大文档搜索时的同步扫描开销。

### 修复

- 修复从编辑模式切回预览模式后，旧的预览搜索内容和高亮可能残留的问题。

## [0.2.0] - 2026-06-26

### 新增
- 新增多标签页：可同时打开多个 Markdown 文件，工具栏下方水平标签栏显示文件名、未保存圆点和关闭按钮。
- 标签支持点击切换、中键关闭，再次打开已打开的文件会聚焦到对应标签而非重复打开。
- 每个标签独立保留原始内容、未保存草稿、编辑/预览模式、大纲和滚动位置，切换标签即恢复现场。
- 应用重启后恢复上次打开的标签列表与激活项（不持久化未保存草稿），并迁移旧的单文件恢复记录。
- 文件外部变更按路径分发到对应标签：非活动且无未保存修改的标签静默重载，活动或脏标签弹出冲突提示。
- 关闭带未保存修改的标签或退出窗口时，逐个弹出三选项确认（保存/不保存/取消）。
- 新增 CodeMirror Markdown 源码编辑模式，支持预览/编辑切换、行号、折叠、括号匹配、查找/替换和跳转行。
- 新增手动保存与另存为，`Ctrl+S` 改为保存当前文件，`Ctrl+Shift+S` 用于另存为。
- 新增未保存修改保护：切换文件、关闭窗口、关闭文件夹和外部文件变更时提示保存、放弃或取消。

### 修复
- 修复接入未保存修改保护后，点击窗口右上角关闭按钮无法退出应用的问题：补充 `core:window:allow-destroy` 权限，并在无未保存修改时不拦截关闭、直接走系统默认关闭，仅在有未保存修改时拦截并弹窗确认。

### 工程
- 新增 `vendor-codemirror` Vite 分包，隔离 CodeMirror 相关依赖。
- 在 `capabilities/default.json` 增加 `core:window:allow-destroy`，使前端确认未保存修改后可调用 `destroy()` 退出窗口。

## [0.1.2] - 2026-06-25

### 新增
- 支持界面中英双语切换：跟随系统语言，顶部工具栏 `中/EN` 按钮手动切换，持久化记忆。
- GitHub README 支持中英双语：默认英文 README.md，中文 README.zh-CN.md，顶部互相跳转。
- 暗色模式下原生窗口标题栏（Mica 背景）随应用主题同步切换为深色。

### 优化
- 保持浅色主题原有 GitHub 风格配色，仅对暗色主题按高级阅读器方向优化。
- 暗色主题下使用深色顶部栏和文件树、暖灰大纲区、深色纸张式阅读区。
- 暗色主题下文件树当前项为金色左边线高亮，大纲当前项为青绿色左边线高亮。
- 暗色主题下优化 Markdown 正文标题、引用块和代码块排版，提升阅读层级和桌面阅读器质感。
- 修复暗色主题下 Vue scoped CSS 选择器错误编译导致大纲/文件树高亮不生效的问题，改为 CSS 变量控制主题样式。

### 工程
- i18n 文案转为类型安全：以 zh-CN 为 MessageSchema 源，en-US 用 `satisfies MessageSchema` 保证两套语言 key 一致。
- MarkdownView 渲染流程拆分：纯主题切换走 `refreshThemeRender` 仅强制 Mermaid 重绘，避免完整 DOM 重建。

## [0.1.1] - 2026-06-25

### 修复
- 修复 PDF 导出偶发 `Edge 执行完成但 PDF 未生成` 的问题：Edge headless 现在会最多重试 3 次，并为每次尝试使用独立临时 profile。
- 增强 PDF 导出的 Edge 稳定性参数，降低 Edge 首次启动、后台限速或 profile 初始化导致空输出的概率。
- PDF 导出失败时保留诊断 HTML，并返回每次尝试的 exit code / stdout / stderr，便于定位用户环境问题。
- 修复暗色主题下滚动条仍为白色的问题，滚动条现在跟随亮/暗主题切换为灰色/深色。
- 修复通过文件关联或命令行打开局域网共享（UNC）路径 Markdown 文件时，`\\?\UNC\...` 被错误裁剪为 `UNC\...`，导致读取失败的问题。
- 修复 UNC 路径转换为 `file://` URL 时的前缀处理，避免后续导出或资源路径处理踩同类问题。

### 工程
- 为 Windows 扩展路径前缀处理新增单元测试，覆盖本地盘路径、普通 UNC 路径和扩展 UNC 路径。

## [0.1.0] - 2026-06-24

首个公开版本，功能完整可用。

### 新增

#### 核心阅读
- Markdown 渲染（CommonMark + GFM）
- 代码语法高亮（highlight.js）
- 数学公式渲染（KaTeX 按需加载）
- 流程图 / 时序图（Mermaid 按需加载）
- 任务列表、脚注、Emoji、标题锚点
- 亮 / 暗主题切换

#### 导航
- 左侧文件树（递归扫描，过滤 .git/.node_modules）
- 右侧 TOC 大纲（滚动同步高亮）
- 三栏可拖拽分隔条，宽度持久化
- 内部链接 `[xxx](./y.md#z)` 跳转
- 图片相对路径自动解析（asset 协议）

#### 查找
- 当前文档查找（Ctrl+F，TreeWalker 遍历文本节点高亮）
- 跨文件全文搜索（Ctrl+Shift+F，Rust 后端）

#### 导出
- 导出 HTML（自包含单文件，图片/CSS 全内嵌）
- 导出 PDF（Edge headless，所见即所得，1-3 秒）
- 导出 DOCX（pandoc）
- 系统打印（Ctrl+P）

#### 系统集成
- 文件关联：`.md / .markdown / .mdx` 默认打开方式
- 单例运行：第二次打开会复用现有窗口
- 命令行参数：支持 `md-reader.exe <file>`
- 拖拽文件到窗口
- 文件变更监听自动刷新（notify-debouncer-mini，300ms 防抖）

#### 体验
- 阅读设置：字号 / 行高 / 页面宽度 / 字体
- 最近文件历史（localStorage，最多 20）
- 每文件独立的滚动位置记忆
- 启动恢复上次最后浏览的文件
- Edge 路径自动探测 + 手动指定持久化

### 工程
- TypeScript 严格模式
- ESLint flat config + Prettier
- Vite 代码分包（Mermaid/KaTeX/hljs 独立 chunk）
- 构建产物：md-reader.exe ~5.6 MB，MSI ~3.8 MB

### 已知限制
- DOCX 导出需要安装 pandoc
- PDF 导出依赖系统 Edge（Win10/11 自带）
- Mermaid 在 DOCX 中可能样式简化（pandoc 转换限制）
